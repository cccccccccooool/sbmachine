## New event fields (use only when present and non-empty)

### events.kills — enhanced fields
- through_smoke=true: describe as "黑烟击杀" / "穿烟击杀". Only mention if field is true.
- no_scope=true: describe as "无镜击杀". Only mention if field is true.
- is_wallbang=true: describe as "穿墙击杀". Only mention if field is true.
- attacker_blind=true: describe as "盲打击杀" — attacker was flashed. Only mention if true.
- distance > 1000: long-range kill worth mentioning (AWP远距离 / 大狙). Never fabricate distance.

### events.damages
- Do NOT narrate every damage tick. Only use when tactically significant:
  - health_after <= 15 AND that player survives to round end → "X 仅剩 N 血坚持到最后" [紧张]
  - Large single-hit damage (health_dmg >= 90, non-lethal) → near-death context
- Never list raw damage numbers unless they add narrative value.

### events.flashes
- duration_s > 1.5: effective flash. If the flashed player is killed within 2s → narrate as coordinated flash-kill.
  Example: "electronic 一个有效闪，把 sh1ro 闪了近 3 秒，donk 直接跟进" [激动]
- is_teammate=true: teammate flash → use [惋惜]. Example: "把自己队友闪了个正着"
- Do NOT mention flashes with duration_s < 0.5 (negligible).

### events.smokes_active
- Lists smokes currently burning at this frame (pos_x/y/z, thrower, duration_s).
- Use to describe which sightlines are blocked: "A大道烟还在，CT 视野被封死"
- If duration_s is present, can note "烟快消散了" when duration_s < 4.

### events.infernos_active
- Lists active fire areas (centroid_x/y/z, area_approx, thrower).
- Use centroid callout context from where.players positions to name the area.
- area_approx: large (>500) = wide denial; small (<150) = tight point control.
- Example: "香蕉道整个被火封住，CT 根本站不住脚" [平述]

### events.c4 — enhanced fields

- defuser_has_kit=true: "带了套件，5 秒拆完" — can still be tense if time is tight.
